Ischia Transfer Beta - Capacita Attuali

Data documento: 9 marzo 2026

Scopo: quadro sintetico delle funzionalita realmente disponibili oggi per demo commerciale.

1. Piattaforma Core

Architettura

  • Frontend Next.js App Router + TypeScript strict + Tailwind.
  • Backend su Supabase (Auth, Postgres, RLS multi-tenant).
  • Mappa con Leaflet + OpenStreetMap.
  • Deploy target Vercel.

Ruoli e sicurezza

  • Ruoli supportati: admin, operator, driver, agency.
  • RBAC lato app + policy RLS lato database.
  • Isolamento dati per tenant su tabelle business.

2. Operativita Giornaliera

  • Dashboard KPI e monitoraggio stato servizi.
  • Creazione e modifica prenotazioni transfer/bus tour.
  • Dispatch: assegnazione autista e mezzo.
  • Vista driver mobile-first con avanzamento stato servizio.
  • Mappa operativa con hotel e servizi.

3. Modulo Hotel Ischia (OSM)

  • Import hotel da Overpass API (OpenStreetMap) via endpoint admin.
  • Campi gestiti: nome, nome normalizzato, indirizzo, citta, zona, coordinate, fonte OSM, stato attivo.
  • Deduplica per source_osm_type + source_osm_id e fallback nome normalizzato + coordinate.
  • Fallback robusto query Overpass (area + bbox Ischia) con endpoint multipli.
  • Caching import per ridurre chiamate esterne e mantenere stabilita operativa.
  • Gestione manuale in UI: modifica dati, attiva/disattiva, alias hotel, evidenza record incompleti.

Tabelle dedicate: hotels, hotel_aliases, hotel_import_runs.

4. Ingestion e Automazioni

  • Import email inbound e conversione verso draft service.
  • Parser PDF/email per supporto estrazione dati operativi.
  • WhatsApp Cloud API: invio reminder, webhook stato, job cron.
  • Configurazione tenant WhatsApp da pannello admin.

5. Pricing e Reportistica

  • Modulo pricing con regole, override e storico.
  • Export servizi e analytics (endpoint dedicati, incluso formato Excel).

6. Stato Qualita Tecnica

  • Build applicativa: OK.
  • Lint: OK (presenti warning legacy non bloccanti nel client storico).
  • Validazione payload API tramite Zod nei punti sensibili.

7. Limiti Operativi da esplicitare in demo

  • Overpass/Nominatim sono servizi pubblici: possibili timeout, rate limit, risposte incomplete.
  • Dati OSM non sempre perfetti: alcuni hotel possono richiedere correzione manuale.
  • Per questo il sistema salva su DB e consente revisione operatore/alias.

Messaggio commerciale sintetico: la piattaforma e gia utilizzabile per flusso end-to-end (prenotazione, dispatch, driver, reminder, export) con nuovo asset hotel localizzato per Ischia e governance dati lato operatore.
